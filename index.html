Dein erster KI-Agent: Postfach auf Autopilot — von Blueprint bis Betrieb

Lerne wie du in n8n einen echten E-Mail-Agenten baust — von der ersten Node bis zum produktiven Betrieb.

Für alle, die gerade eine KI-Agentur aufbauen möchten.

  • n8n
  • Groq API

Warum dieser Kurs

  • Keine Theorie — echter Workflow: live gebaut, Schritt für Schritt nachvollziehbar.
  • Kostenlos umsetzbar: Groq Free Tier, n8n Self-Hosted — kein Abo-Zwang für den ersten Agenten.
  • Skalierbar: dasselbe Muster funktioniert für 80% aller Automatisierungen.

Kursstruktur

  1. 01

    Blueprint: Bevor du baust — wie dein Agent denkt und entscheidet

    Lernziel: Du hast deinen eigenen Agent-Blueprint — die vier Komponenten, ihre Verbindungen und die Kategorien, die dein Agent erkennen soll.

    Aufbau

    1. Das Problem — warum manuelles Email-Triaging verschwendete Kapazität ist.
    2. Der Agent als System: Trigger, Analyse, Routing, Antwort.
    3. Tool-Entscheidung — warum n8n + LLM API (kein teures Setup, sofort einsetzbar).
    4. Deinen Agent-Blueprint erstellen (5 Minuten, Stift oder Miro).
  2. 02
    Live Demo

    Build: Von Null zum laufenden Agenten — live in n8n

    Lernziel: Du hast einen laufenden n8n-Workflow, der eingehende Emails empfängt, per LLM klassifiziert und automatisch mit einer ersten Antwort reagiert.

    Aufbau

    1. Workflow-Überblick: die 5 Nodes in einfacher Sprache.
    2. IMAP Trigger Node einrichten (Credentials, Host, SSL).
    3. HTTP Request Node: LLM klassifiziert Email in ANFRAGE / BESCHWERDE / SPAM / SONSTIGES.
    4. Edit Fields Node: Kategorie per Drag & Drop extrahieren.
    5. Switch Node: 4 Routing-Regeln aufbauen.
    6. Send Email Node: Kategorie-Tag im Subject + automatische Antwort.
    7. Live-Test: Email schicken, Postausgang prüfen.
    ▶️ Video ansehen
  3. 03
    Live Demo

    Launch: Dein Agent antwortet selbst und wird messbar

    Lernziel 1: Du kannst einen LLM-Prompt so strukturieren, dass er kategorie-abhängig unterschiedliche Tonalitäten produziert — ohne mehrere separate Nodes zu bauen.

    Lernziel 2: Du hast einen automatischen Aktivitäts-Log eingerichtet, mit dem du oder dein Kunde jederzeit nachvollziehen kann, was der Agent wann getan hat.

    Aufbau

    1. Kurzrückblick: Was noch fehlt — statische Antworten vs. individuelle Reaktionen.
    2. LLM-Node vor den Switch: Warum ein Node für alle Kategorien reicht.
    3. System-Prompt bauen: Kategorie + E-Mail-Inhalt als Variablen.
    4. Switch Node und Send Email aktualisieren.
    5. SPAM Branch: Warum keine Antwort die bessere Entscheidung ist.
    6. Google Sheets einrichten und Node mappen ("Copilot" / "AutoEmail").
    7. End-to-End Test + Ausblick auf Video 4.
    ▶️ Video ansehen
  4. 04

    Monitor: Wie du weißt ob dein Agent läuft — und was du tust, wenn nicht

    Lernziel: Du weißt, wie du erkennst ob der Agent korrekt läuft, wie du Fehler systematisch abfängst und wie du ihn nach dem Launch betreust.

    Aufbau

    1. Wann du eingreifen musst und wann der Agent es selbst regelt.
    2. n8n Execution Log: was du täglich im Blick haben solltest.
    3. Error Trigger Node einrichten: automatische Benachrichtigung wenn etwas schiefläuft.
    4. Typische Fehlerquellen und wie du sie löst (IMAP-Timeout, Rate Limit, leere Email).
    5. DSGVO-Check: Was du vor dem Produktiveinsatz beim Kunden klären musst.

    Faustregel: Kunde im EU-Raum — das Thema ansprechen, bevor der Workflow live geht.
